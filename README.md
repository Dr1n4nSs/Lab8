# Сыпачев Иван КМБ-2 Лабараторная №8
# Задание 1
## Задача 10
### Текст задачи
Разработать консольное приложение с дружественным интерфейсом с возможностью выбора
заданий для работы с «базой данных (БД)», хранящейся в бинарном файле. Перечень полей
(минимум 5), достаточно полно характеризующих заданную в варианте предметную область,
предложить самостоятельно (постараться отразить в перечне полей такие, которые требуют разных
типов данных). Приложение должно выполнять следующие функции:
1. Чтение базы данных из бинарного файла.
2. Просмотр базы данных.
3. Удаление элементов (по ключу).
4. Добавление элементов.
5. Реализация 4 запросов (формулировки запросов придумать самостоятельно). 2 запроса
должны возвращать перечень, 2 запроса одно значение.
В классе должны присутствовать свойства, конструкторы, перегруженный метод ToString(). Весь
функционал приложения реализовать в виде методов вспомогательного класса с помощью LINQзапросов
### Алгоритм решения
Интерактивное меню: Реализовано через бесконечный цикл while(true) с обработкой пользовательского выбора через оператор switch. Включает механизмы просмотра, добавления, удаления записей и вызова аналитических отчетов.
Обработка исключений: Основной цикл обернут в блок try-catch, что позволяет программе не завершаться аварийно при некорректном вводе данных (например, при ошибке парсинга чисел), выводя детали ошибки пользователю.
Сервисный слой: Для загрузки и сохранения базы данных используется статический класс MuseumService, что отделяет логику хранения от интерфейса.
Валидация при добавлении: При вводе нового экспоната программа выполняет многоуровневую проверку:
Уникальность: Проверяет через database.Any(), не занят ли уже данный инвентарный номер.
Корректность параметров: Год создания должен быть в диапазоне от 1 до 2026, а цена — строго положительной.
Удаление: Осуществляется поиск и удаление объектов по инвентарному номеру с использованием метода RemoveAll.
Фильтрация: Метод GetAntiqueExhibits отбирает предметы, созданные ранее указанного года (например, 1800 г.).
Сложная выборка: Метод GetTopExpensiveRestored извлекает наиболее ценные объекты, прошедшие реставрацию.
Вычисление средней оценочной стоимости всего фонда (GetAverageValue).
Подсчет количества особо древних экспонатов (старше 1500 года) через метод CountAncientExhibits.
Использует методы Console.Clear() для поддержания чистоты экрана и Console.ReadKey() для задержки вывода, позволяя пользователю изучить результаты запросов перед возвратом в главное меню.
Вывод данных об объектах реализован через делегат database.ForEach(e => Console.WriteLine(e)), что предполагает переопределенный метод ToString() в классе Exhibit.
Объект XmlSerializer: Используется для автоматического преобразования списка объектов Exhibit в формат XML и обратно. Это позволяет сохранять состояние базы данных между запусками приложения.
Метод SaveDatabase: Выполняет запись текущей коллекции в файл. Перед началом работы вызывает проверку пути, после чего создает поток FileStream в режиме перезаписи (FileMode.Create).
Метод LoadDatabase: Реализует восстановление данных. Если файл по указанному пути отсутствует, метод возвращает новый пустой список, предотвращая ошибки при первом запуске программы.
Метод GetAntiqueExhibits: Выполняет фильтрацию коллекции по году создания и последующую сортировку результатов в хронологическом порядке (от самых старых к более современным).
Метод GetTopExpensiveRestored: Формирует выборку из трех самых дорогих экспонатов, имеющих статус «отреставрирован». Использует цепочку методов Where → OrderByDescending → Take.
GetAverageValue: Вычисляет среднюю стоимость экспонатов. Содержит проверку Any(), чтобы избежать ошибки деления на ноль в случае пустой коллекции.
CountAncientExhibits: Подсчитывает общее количество предметов, созданных ранее 1500 года.
Метод ValidateFile: Служит защитным механизмом, проверяя наличие целевого файла. В случае неудачи выводит уведомление в консоль и возвращает логический флаг, блокирующий дальнейшие операции ввода-вывода.
